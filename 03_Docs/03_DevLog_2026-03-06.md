# DevLog - 6 Mars 2026: Déploiement HF Space + TREC IR Metrics

## Résumé des modifications

### 1. HuggingFace Space Configuration

**Problème initial:**
- Le Space HF ne démarrait pas à cause de permissions manquantes sur `/app/instance`
- Le fichier `.env` n'était pas trouvé

**Solutions appliquées:**
- Modification du Dockerfile pour ajouter `chmod 777 /app/instance`
- Correction du chemin `sys.path` dans `app.py` (`/app` au lieu de `/home/user/...`)
- Mise à jour du CMD pour utiliser `app:app` au lieu de `syscred.backend_app:app`

### 2. TREC AP88-90 Corpus

**Fichiers créés:**
- `syscred/convert_trec.py` - Script de conversion gz → JSONL
- `trec_corpus.jsonl` - 242,929 documents (650MB)

**Problème de déploiement:**
- Le fichier JSONL est stocké avec git-lfs mais le container HF ne le télécharge pas correctement
- Le fichier fait 134 bytes (pointeur LFS) au lieu des 650MB réels
- **Solution de contournement:** Utilisation d'un corpus de démonstration de 7 documents pour les tests

### 3. Métriques IR Implémentées

**Endpoints disponibles:**
- `POST /api/trec/search` - Recherche avec BM25/TF-IDF/QLD
- `POST /api/trec/metrics` - Calcul des métriques
- `GET /api/trec/health` - Statut du système TREC
- `GET /api/trec/corpus` - Info sur le corpus

**Métriques calculées:**
- Precision@K
- Recall@K
- MAP (Mean Average Precision)
- NDCG (Normalized Discounted Cumulative Gain)
- MRR (Mean Reciprocal Rank)

### 5. Commandes Utiles

```bash

python syscred/convert_trec.py trec_ap88_90/ trec_corpus.jsonl

# Tester l'API TREC
curl -X POST https://domloyer-syscred.hf.space/api/trec/search \
  -H "Content-Type: application/json" \
  -d '{"query": "climate change", "k": 5}'

# Vérifier le health
curl https://domloyer-syscred.hf.space/api/trec/health
```

### 6. Issues Connues

1. **Corpus LFS non chargé** - Le fichier trec_corpus.jsonl (650MB) n'est pas téléchargé dans le container HF
   - Cause: git-lfs non installé dans le container
   - Impact: Utilise le corpus demo (7 documents)
   - Solution possible: Modifier le Dockerfile pour installer git-lfs

2. **Temps de réponse long** - Le chargement du corpus complet prendrait ~30s
   - Solution: Limiter à 10k documents pour les tests
